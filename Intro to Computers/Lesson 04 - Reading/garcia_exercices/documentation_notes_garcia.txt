Exercise 4: Library Documentation Deep Dive
1. Finding the function: https://requests.readthedocs.io/en/latest/api/#requests.get
2. url, params, **kwargs parameters are accepted; only URL is required, with the rest optional
3. It returns a Response object, with text/content/encoding/apparent_encoding attributes (and more); methods include close(), iter_content(), iter_lines(), json(), and raise_for_status()
    To get the response body as text, access the .text attribute
    To get the HTTP status code, access the status_code attribute
4. Examples:
    Pass query parameters:
        payload = {'key1': 'value1', 'key2': 'value2'}
        r = requests.get('https://httpbin.org/get', params=payload)
    Handle authentication:
        rom requests.auth import HTTPBasicAuth
        basic = HTTPBasicAuth('user', 'pass')
        requests.get('https://httpbin.org/basic-auth/user/pass', auth=('user', 'pass'))

requests.get() Documentation Summary:

Location in docs: https://requests.readthedocs.io/en/latest/api/#requests.get
Required parameters: url
Optional parameters: params, **kwargs
Returns: Response object
Common use cases: to retrieve data from a web server or API endpoint without modifying the server's state
Related functions: requests.post(), requests.put(), requests.patch(), requests.delete(), requests.head(), etc.

Practical Application:
import requests
import json

url = 'https://api.github.com/users/github'
response = requests.get(url)
print(f"Status Code: {response.status_code}")

json_data = response.json()
print(json.dumps(json_data, indent=4))

Comparing Versions:
dict.update() has the same functionality in Python 3.8 and Python 3.11, but |= other was added in 3.9
